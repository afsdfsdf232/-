有些浏览器在请求时，添加一个非标准字段 Connection: keep-alive (一个可复用的 TCP 连接就建立了，直到客户端或者服务器主动关闭连接。但是，这不是标准字段，不同实现的行为可能不一致，因此不是根本的解决方案)

HTTP 1.1 版本引入了持久连接，即 TCP 连接默认不关闭，可以被多个请求复用，不用声明 Connection: keep-alive （客户端和服务端发现对方一段时间没有活动，就可以主动关闭连接。不过，规范的做法是，客户端在最后一个请求时，发送 Connection：close，明确要求服务器关闭 TCP 连接。目前，对于同一域名，大多数浏览器允许同时建立6个持久连接）

管道机制：HTTP 1.1 引入了管道机制，即在同一个 TCP连接里面，客户端可以同时发送多个请求。这样就改善了HTTP协议的效率。（从前发送请求后需等待并接收响应，才能发送下一个请求。管道机制出现后，不用等待响应即可直接发送下一个请求。这样就是同时并行发送多个请求，从而提升连接的效率。）
